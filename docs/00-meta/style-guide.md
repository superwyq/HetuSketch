# 文档写作规范

## 命名

- 目录使用数字前缀表达层级，例如 `20-architecture/`。
- 文件使用 kebab-case，例如 `system-overview.md`。
- ADR 使用 `ADR-序号-主题.md`，例如 `ADR-001-why-electron.md`。

## 内容结构

- 优先回答“为什么、是什么、怎么做、边界是什么”。
- 模块文档按“职责 → 依赖 → 核心文件 → 数据流 → 对外接口 → 已知问题”组织。
- 接口文档必须说明调用方、入参、返回值、错误/降级行为。

## 链接规范

- 文档引用代码使用相对路径，例如 `../../src/shared/ipc.ts`。
- 代码引用文档建议使用 `@see docs/...` 注释。
- 同类概念统一使用术语表中的名称。
