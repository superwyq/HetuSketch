# ADR-002: 使用 SQLite / FTS5 作为本地索引层

## 状态

已接受（Accepted）

## 背景

项目需要对本地 JSON/Markdown 设定文件进行快速搜索、最近访问、统计、RAG 分块和配置持久化，同时保持用户数据可读、可备份。

## 考虑的选项

| 方案 | 优点 | 缺点 |
| --- | --- | --- |
| 仅文件系统 | 数据透明，易备份 | 搜索、统计和增量同步性能差 |
| SQLite + FTS5 | 单文件数据库、可嵌入、搜索性能好、可重建 | 需维护索引同步 |
| 外部搜索服务 | 检索能力强 | 不符合本地优先，部署复杂 |

## 决策

使用 better-sqlite3 + SQLite FTS5 作为派生索引层，JSON/Markdown 仍为事实源。

## 后果

- 正面影响：本地搜索和统计能力稳定，索引可删除重建。
- 负面影响：需要处理文件变更监听、增量同步和 schema 演进。
- 后续跟进：增加 schema 版本化迁移，评估 sqlite-vec 或 ANN 索引优化向量检索。
