# 作者工作流用户故事

## 建立创作项目

作为长篇小说作者，我希望创建作品、书目、分卷和章节，以便按长期创作结构组织正文。

验收要点：

- 可创建作品并设为当前作品。
- 可创建书目并绑定设定集。
- 可创建分卷和章节，并在章节树中查看。

## 录入设定

作为创作者，我希望结构化保存角色、世界观和伏笔，以便写作时快速检索并参与校验。

验收要点：

- 角色支持红线、性格、能力、背景等字段。
- 世界观支持类别和规则。
- 伏笔支持 open/resolved/abandoned 状态。

## 编排剧情画布

作为长篇架构型作者，我希望从章节进入剧情画布，把角色、世界观、线索和章节素材组织成可视化事件链，以便在写正文前确认顺序、因果和状态变化。

验收要点：

- 首次从章节进入剧情画布时自动创建空画布，再次进入加载已有画布。
- 可双击创建剧情卡，从素材库拖拽或点击绑定角色、世界观、线索、章节和模板。
- 可拖拽卡片、连线、框选、缩放和平移，并保存视口。
- 可使用三幕式、推理揭示链、群像交叉线模板快速搭建结构。

## 生成与结算正文

作为作者，我希望 AI 只根据我确认的剧情卡和设定生成章节正文，并把状态变化作为可审核建议，以便保持故事主导权。

验收要点：

- 支持单卡、选区、全章、续写和重写生成。
- 生成前保留旧章节正文快照，生成后正文写入章节 Markdown。
- 未配置 AI 时可以降级生成结构化 Markdown 草稿。
- State Diff 不会自动写入状态快照，用户可逐条确认、修改或拒绝。

## 校验剧情一致性

作为悬疑、群像或同人作者，我希望在画布阶段发现时间线、角色状态、世界规则和伏笔顺序问题，以便减少正文返工。

验收要点：

- 校验结果按时间线、角色状态、行为红线、世界规则、伏笔顺序和章节衔接分类展示。
- 点击校验结果可定位到剧情卡；有生成正文时显示 Markdown 段落定位。
- 线索被回收后，系统提示是否将对应伏笔条目更新为 resolved。

## 写作与校验

作为作者，我希望在写作过程中检查文本是否违反已有设定，以便及时修正逻辑问题。

验收要点：

- 可选择当前章节、分卷或作品范围校验。
- 基础校验离线可用。
- AI 增强不可用时明确显示降级状态。

## 外部写作辅助

作为使用 Word 或浏览器写作的作者，我希望通过置顶悬浮窗查询设定，以便无需切换主应用。

验收要点：

- `Ctrl+Shift+H` 可切换悬浮速查窗。
- 悬浮窗可搜索并显示最近访问。
- 置顶状态可切换。
