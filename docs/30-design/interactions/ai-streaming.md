# AI 流式交互

## 使用场景

- AI 增强校验。
- RAG 问答。
- 设定补全。
- 伏笔提醒。

## 交互模型

1. 用户点击 AI 相关按钮。
2. UI 创建或复用 `requestId`。
3. Preload 监听 `chunk/end/error` 临时通道。
4. 主进程逐块发送 `AiStreamChunk`。
5. UI 展示实时状态、证据和最终结果。
6. 结束后移除监听器。

## 降级展示

- `degraded`：显示本地结果与“AI 增强不可用”提示。
- `partial`：展示可用部分和缺失原因。
- `error`：展示错误摘要，不暴露 API Key 或敏感请求内容。

## 设计约束

- AI 结果必须标注为建议，不自动覆盖正文或设定。
- 结构化 evidence 应可点击跳转相关条目。
- 长文本上下文被裁剪时应提示用户缩小范围。
